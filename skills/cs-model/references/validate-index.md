# Validate model truth and indexing

Challenge the edit for currency, authority, terminology, observability, conflicts, supersession, sensitive data, broken links and index size. Confirm that the named consumer can discover and use the artifact.

Use Harness-backed diff/format/link or targeted contract checks according to risk. A documentation-only edit may still be L2/L3 when it changes a public, persistent, authorization or safety control surface.

Do not complete because a file exists. Required evidence must pass, blocking conflicts must be resolved, and the Harness must grant completion. Promote reusable knowledge only when a future Agent, checker, runtime, reviewer or human decision process has a concrete consumption path.
